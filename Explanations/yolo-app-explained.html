How YOLOv8 Works - You Only Look Once
Single-pass detection, 80 classes, 5 model sizes
▼

YOLO (You Only Look Once) revolutionised object detection by processing the entire image in a single forward pass. Older methods used sliding windows (slow) or region proposals (two-stage). YOLO predicts all bounding boxes and class probabilities simultaneously, making it fast enough for real-time detection.

All five model sizes are selectable from the sidebar. The model is pre-trained on COCO (Common Objects in Context) - 80 classes including people, vehicles, animals, furniture, electronics, food, and more.

📌 NMS (Non-Maximum Suppression): When YOLO detects the same object multiple times with overlapping boxes, NMS keeps only the best one. The IoU threshold (adjustable in the sidebar) controls how much overlap is allowed before a duplicate is removed.

Features Deep Dive
Heatmaps, preprocessing, video tracking, batch processing, export
▼

Detection Heatmap

Instead of bounding boxes, the heatmap mode creates a confidence-weighted density map. Each detection's bounding box area is filled with its confidence score, then the whole map is smoothed with a Gaussian filter (σ=20) and overlaid on the original image using a JET colourmap. Red = high detection density, blue = low. This is useful for understanding spatial patterns — like "most people cluster near the door."

Image Preprocessing

The sidebar offers brightness, contrast, and sharpness sliders (0.5× to 2.0×). These adjust the image before detection using PIL's ImageEnhance. This helps with poor-quality uploads — a dark photo can be brightened to help YOLO detect objects it would otherwise miss.

Video Detection + Object Tracking

The video tab processes uploaded videos frame-by-frame. With tracking enabled, YOLO assigns persistent IDs to objects across frames — so "Person #3" stays "Person #3" even as they move. The app generates a "Detections per Frame Over Time" line chart, an annotated video download, and a CSV of all detections with frame numbers and track IDs.

Batch Processing

Upload multiple images at once. Each gets individual detection results, then an aggregate summary combines all detections into a single bar chart and downloadable CSV. Useful for analysing a whole folder of images at once.

Export Options

Every detection can be exported in three formats: CSV (for spreadsheets), JSON (for APIs), and annotated PNG (for sharing). The DataFrame includes class name, confidence, bounding box coordinates (x1, y1, x2, y2), and box dimensions (width, height).

Summary — Key Takeaways
▼
🎯 Takeaway #1 YOLO = Real-Time Detection. YOLOv8 processes an entire image in a single forward pass, detecting all 80 COCO classes simultaneously with bounding boxes and confidence scores. No sliding windows, no region proposals.
🎯 Takeaway #2 Streamlit Makes ML Apps Easy. The entire web app (upload, detect, display, chart, export) is built in a single Python file using Streamlit's declarative API. No frontend skills needed.
🎯 Takeaway #3 Caching Is Critical for Performance. @st.cache_resource ensures the YOLO model loads only once, not on every user interaction. Without this, the app would freeze for 2–3 seconds on every click.
🎯 Takeaway #4 Video Tracking Adds a Dimension. YOLO's built-in tracking (model.track()) assigns persistent IDs across frames, turning simple detection into object tracking. This enables counting unique objects and analysing movement patterns.
🎯 Takeaway #5 DataFrames Enable Flexibility. Converting YOLO's raw output into a Pandas DataFrame allows for powerful filtering, charting, and export options. The structured format is key to building a user-friendly interface.
